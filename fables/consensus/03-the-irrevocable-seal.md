# The Irrevocable Seal

> *"在文件上盖章只是开始。等到章上的墨迹渗入纸纤维，才算真正生效。"*

## 故事

云州城的文书系统在盖章后还有一个传统：等待。

---

**即时章与最终章**

旧制度下，文书盖章后立即生效。但有一个问题：如果后来发现章是伪造的，或者盖章的人后来被证实没有权限，已经生效的文书怎么办？

答案是：撤回。但撤回已经执行的文书极其困难——比如已经完成的交易、已经转移的财产，如何撤销？

新制度引入了两种章：

**即时章**：盖章后文书立即可以被引用、被阅读。但它有一个隐藏的属性——它可能被"替换"。如果在后续的检查中发现问题，即时章可以被一个更正后的文书替换。

**最终章**：文书被盖了足够多的章（来自不同官员的确认），并且经过了足够长的时间，它就变成了"不可撤销"的。最终章一旦被盖，文书永远生效，没有任何力量可以修改或撤回。

---

**从即时到最终**

文书从即时章到最终章的过程是这样的：

1. 第一个官员盖章（提议）。文书进入"即时"状态。
2. 接下来的一段时间里，其他官员陆续检查文书，如果同意，就盖上自己的确认章。
3. 当确认章的数量达到阈值（比如总官员数的 2/3），文书就进入"最终"状态。

关键在于第三步的"一段时间"。在这段时间里，有可能出现两个互相矛盾的文书都被部分官员确认的情况。比如：

- 文书 A 被 40% 的官员确认。
- 文书 B（与 A 矛盾）被 35% 的官员确认。

如果系统现在停止，A 和 B 都没有达到 2/3 的阈值，所以都不会最终生效。官员们会继续确认，直到其中一个达到阈值。

但如果有一个恶意官员集团，他们同时确认了 A 和 B（双重确认），试图让系统混乱怎么办？

答案是：双重确认的官员会被发现并惩罚——他们的确认资格会被取消，之前他们的所有确认都失效。这样，系统最终会收敛到一个唯一的最终文书。

---

**概率 vs 确定性**

云州城的旧账本（基于算力竞争）有一个不同的最终性模型：

在旧制度下，文书盖章后，它"很可能"是正确的，但不是"绝对"正确。理论上，如果有人拥有超过一半的算力，他们可以重写历史——替换已经被盖章的文书。但随着文书被越来越多的后续文书覆盖，重写的成本呈指数级增长。

在新制度下（基于押注共识），最终性是确定性的：一旦文书被足够多的官员确认并达到最终状态，它就绝对不可更改。不需要概率估算，不需要假设"没有人拥有超过一半的算力"。

用数字来说：
- 旧制度：一笔交易在 6 个区块后"几乎不可能"被回滚。但"几乎不可能"不是"绝对不可能"。
- 新制度：一笔交易在大约 12 分钟后（2 个 epochs）达到最终状态。一旦最终，绝对不可回滚。

---

**回滚的幽灵**

即使在新制度下，在最终章盖下之前，仍然可能发生"回滚"。

如果一个官员的节点出现了软件 bug，或者网络分区导致他看到了一个与其他官员不同的世界，他可能会确认一个"错误的"文书。在这种情况下，系统会短暂地出现分叉——两个版本的账本同时存在。

但这种分叉是暂时的。当网络恢复、官员们重新同步后，他们会根据规则选择"权重更大"的版本继续，抛弃另一个版本。在分叉期间被确认的文书，如果最终没有被选中，就会失效。

这就是为什么"即时章"不等于"最终章"。在即时章盖下后、最终章盖下前，一切都还有变数。

---

多年后，云州城的商人们形成了一个习惯：小额交易信任即时章，大额交易等待最终章。一位老账房先生说："即时章让你跑得更快，最终章让你睡得更安稳。聪明的交易者知道什么时候该求快，什么时候该求稳。"

---

## 这则寓言在说什么

这则寓言对应的是 **区块最终性（Block Finality）**——区块链中交易被确认后不可回滚的属性。PoW（如比特币）提供的是"概率最终性"，而 PoS（如以太坊）通过 Casper FFG 提供"确定性最终性"。

### 关键映射

| 故事元素 | 技术概念 | 说明 |
|---------|---------|------|
| 即时章 | Block Inclusion / 区块确认 | 交易被打包进区块，可以被引用，但尚未最终 |
| 最终章 | Finalized Block | 通过 Casper FFG 等最终性小工具，区块被足够多的验证者确认后不可回滚 |
| 官员确认 | Validator Attestation | 验证者对区块的有效性进行投票确认 |
| 2/3 阈值 | 超级多数（Supermajority） | 以太坊 PoS 中，需要 2/3 的验证者投票确认，区块才能达到最终性 |
| 双重确认 | Double Vote / 可罚没行为 | 验证者对矛盾的两个区块都投票，会被 slash |
| 惩罚取消资格 | Slashing | 双重投票的验证者会被惩罚，其投票失效 |
| 收敛到唯一文书 | 共识的最终性 | 系统保证最终会收敛到一个唯一的、一致的最终状态 |
| 旧制度概率最终性 | PoW Probabilistic Finality | 比特币等。交易被更多区块覆盖后，回滚的概率指数级降低，但理论上永远不达到 100% |
| 新制度确定性最终性 | PoS Deterministic Finality | 以太坊 PoS。2 个 epochs（约 12.8 分钟）后，区块绝对不可回滚 |
| 6 个区块后 | Bitcoin 6 confirmations | 比特币中，6 个区块确认后认为交易"足够安全"（约 1 小时） |
| 12 分钟后 | Ethereum PoS Finality | 以太坊 PoS 中，约 12.8 分钟（2 epochs）后交易最终化 |
| 分叉与回滚 | Reorg / Fork | 在最终性达到前，可能出现临时分叉，较短的链会被抛弃 |
| 网络分区 | Network Partition | 网络分裂导致部分验证者看到不同的状态，可能产生分叉 |
| 小额信任即时章 | 日常使用接受确认即可 | 普通转账在被打包进区块后就可认为有效 |
| 大额等待最终章 | 大额交易等待最终性 | 交易所存款、跨链桥等通常等待多个确认或最终性 |

### 为什么是这个故事？

最终性是区块链安全的核心保证，但"概率最终性"和"确定性最终性"的区别对很多用户来说很模糊。

用"即时章 vs 最终章"来隐喻最终性，有几个直觉优势：
1. **两阶段确认**：即时章（确认）→ 最终章（最终化）= 交易的两个阶段
2. **概率 vs 确定**：旧制度"几乎不可能"回滚 vs 新制度"绝对不可能"回滚
3. **分叉的临时性**：官员看到不同世界 = 网络分区导致的临时分叉
4. **实际应用**：小额求快、大额求稳 = 不同场景对最终性的不同需求

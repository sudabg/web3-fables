# The Programmable Bazaar

> *"市集不仅卖东西，还允许你在每个关键节点插入自己的规矩。"*

## 故事

云州城的自动市集（Uniswap V2/V3）已经运转了很多年。它的规则是固定的：买卖双方按照恒定乘积公式交易，手续费按比例分配给注池人。没有人可以改变这些规则。

但有一天，市集的建造者宣布了一个革命性的升级："可编程市集"。

---

**市集的核心变革**

在旧市集里，每种商品组合（丝绸-茶叶、大米-银子）都需要建一个独立的市集。云州城里有几千种商品，意味着有几千个独立的市集，每个都有自己的天平、自己的注池、自己的账本。

可编程市集把所有这些合并成了一个"超级市集"。所有商品组合共享同一个天平、同一个账本。不同的是：每个商品组合可以安装自己的"插件"。

插件可以在以下关键时刻介入：

1. **交易前**：当有人想要交易时，插件可以先检查条件——"这个交易者是否在黑名单上？""今天的交易量是否已经超过限额？"

2. **价格计算时**：插件可以修改价格的计算方式——"如果单笔交易超过一千两，手续费从 0.3% 提高到 1%。"

3. **交易后**：插件可以触发后续操作——"完成交易后，把手续费的 10% 自动捐赠给慈善基金。"

4. **注池人加入时**：插件可以设定准入条件——"只有持有特定徽章的人才能成为注池人。"

---

**插件的创造力**

可编程市集的插件系统催生了很多前所未有的创新。

**动态手续费**：
一个插件让手续费根据市场波动自动调整。当市场平静时，手续费很低，吸引交易。当市场剧烈波动时，手续费自动提高，保护注池人免受无常损失。

**限价单**：
另一个插件实现了"限价单"功能。交易者可以设定："当丝绸价格跌到 80 两/匹时，自动买入一百匹。"在旧市集里，这需要不断手动检查价格。在新市集里，插件自动监控并执行。

**时间加权平均价**：
第三个插件在每次交易时记录价格，维护一个"过去一小时的平均价"。其他合约可以引用这个平均价，而不需要自己计算——这大大节省了 gas。

**自定义曲线**：
最激进的插件完全抛弃了恒定乘积公式。一个插件实现了"稳定币曲线"——当两种资产的价格接近 1:1 时，交易几乎没有滑点。这让稳定币之间的兑换变得极其高效。

---

**插件的风险**

但插件也引入了新的风险。

一个恶意的插件可以在交易前偷偷修改价格，让交易者以极不利的价格成交。或者，插件可以在交易后把手续费转到插件作者自己的口袋，而不是给注池人。

市集的建造者意识到了这个问题。他们设计了一个规则：

- 插件的代码是公开的，任何人都可以审计。
- 但安装插件需要"市集管理员"的批准。管理员不审查插件的"创意"，只审查插件是否"安全"——即不会偷钱、不会破坏市集的基本功能。
- 一旦插件被批准安装，它的行为就是确定性的、不可更改的。管理员也不能事后修改插件。

这个设计平衡了"创造力"和"安全性"：任何人都可以提出插件，但只有经过安全审查的插件才能被安装。而且一旦安装，插件就不再受任何人的控制——它按照代码自动执行。

---

**超级市集的账本革新**

可编程市集的另一个重大变化是"瞬时记账"。

在旧市集里，每笔交易都会立即改变账本——天平两边的数量实时更新。这意味着每笔交易都需要写一次账本，消耗一次存储费用。

在可编程市集里，引入了一个"临时账本"。交易者可以在短时间内进行多笔交易，所有这些交易只在临时账本上记录。只有当交易者"结算"时，临时账本的内容才被一次性写入正式账本。

这意味着：一个套利者可以在一次结算中完成十笔交易，但只需要支付一次存储费用。这极大地提高了高频交易的效率。

---

多年后，可编程市集成了云州城最繁荣的商业中心。一位老商人说："旧市集告诉你能做什么。新市集只告诉你不不能做什么——在这两条线之间，是你的创造空间。"

---

## 这则寓言在说什么

这则寓言对应的是 **Uniswap V4 的 Hooks 机制**——V4 最大的创新。V4 将每个交易池（pool）从独立的合约转变为"Singleton"架构下的一个状态条目，并允许在每个 pool 上安装自定义的 Hooks 合约，在交易生命周期的关键点（如 beforeSwap、afterSwap、beforeModifyPosition 等）执行自定义逻辑。

### 关键映射

| 故事元素 | 技术概念 | 说明 |
|---------|---------|------|
| 旧市集几千个独立天平 | Uniswap V3 的 Pool 架构 | 每个 pool 是一个独立合约，部署成本高 |
| 超级市集共享天平 | Uniswap V4 Singleton 架构 | 所有 pools 共享一个合约，通过状态存储区分，部署成本大幅降低 |
| 插件 | Hooks 合约 | 自定义合约，在交易生命周期的特定钩子点被调用 |
| 交易前检查 | beforeSwap hook | 在 swap 执行前调用，可以实施 KYC、限额、黑名单等 |
| 价格计算时 | Dynamic fee hook | 在 swap 过程中可以动态调整手续费 |
| 交易后触发 | afterSwap hook | 在 swap 完成后调用，可以触发捐赠、奖励分配等 |
| 注池人加入时 | before/afterModifyPosition hook | 在流动性添加/移除时调用，可以实施准入控制 |
| 动态手续费 | Dynamic fee tiers | 根据波动率、交易量等动态调整手续费 |
| 限价单 | Limit order hook | 在价格达到阈值时自动执行交易 |
| 时间加权平均价 | TWAP hook | 在每次 swap 时更新累计价格，供外部引用 |
| 自定义曲线 | Custom curves | Hooks 可以实现非恒定乘积的 AMM 曲线（如 Curve 的稳定币公式） |
| 插件审计 | Hooks 的安全审查 | Uniswap V4 的 hooks 由 pool 创建者选择，但用户需要自行审计 hook 的安全性 |
| 瞬时记账 | Flash Accounting | V4 引入"瞬时结算"——多个操作可以先在临时状态中累积，最后一次性结算，节省 gas |
| 临时账本 | Transient Storage (EIP-1153) | V4 使用 EIP-1153 的 transient storage 来实现瞬时记账，操作结束后自动清零 |

### 为什么是这个故事？

Uniswap V4 是 DeFi 协议设计的一个范式转变——从"固定功能的白盒"到"可编程的黑盒"。Hooks 让 AMM 从一个单一的交易机制变成了可定制的平台。

用"可编程市集"来隐喻 V4 Hooks，有几个直觉优势：
1. **插件化**：市集的关键节点插入自定义规则 = Hooks 在交易生命周期中的介入点
2. **Singleton 架构**：共享的超级市集 = V4 将所有 pool 合并到一个合约中
3. **瞬时记账**：临时账本一次性结算 = Flash Accounting 节省 gas
4. **创造力与安全性的平衡**：公开代码 + 管理员审查 = Hooks 的开放性与风险
